# 📚 Педагогическая аналитика

> Локальное desktop-приложение для учителя. Оценки, пробелы, динамика, ИИ-анализ — всё офлайн, всё твоё.

---

## Что это

Инструмент для учителя, который хочет видеть не просто цифры в журнале, а реальную картину: кто застрял на теме, у кого падает динамика, кому нужна помощь прямо сейчас.

<img width="1101" height="752" alt="image" src="https://github.com/user-attachments/assets/2d64509c-6ca9-4034-9390-a82c4866706b" />


Никакого облака. Никакой регистрации. Данные хранятся локально в SQLite.

---

## Возможности

| | |
|---|---|
| 👥 **Ученики** | Добавление вручную, импорт из CSV или Excel |
| 📝 **Оценки** | По темам и предметам, с датой и комментарием |
| 📊 **Аналитика** | Кто отстаёт, пробелы по темам, рейтинг класса |
| 🔍 **Профиль ученика** | Средний балл, лучшие/слабые темы, динамика оценок |
| 🤖 **ИИ-анализ** | Структурированный педагогический разбор без интернета |
| 💡 **XAI-рекомендации** | Советы с объяснением причины и доказательной базой |
| 📄 **Экспорт в PDF** | Профиль ученика, сводка класса, текст ИИ-анализа |

---

## Архитектура

```
pedagog.py          — UI на tkinter + SQLite, главный файл
analytics.py        — DynamicsAnalyzer: анализ динамики оценок по темам
                      ExplainableRec: XAI-рекомендации с обоснованием
hca_pedagog.py      — педагогический мост к HCA / LLM-слою
llm_stub.py         — структурированный генератор анализа (без нейросети)
pdf_export.py       — экспорт отчётов в PDF через reportlab
```

Аналитика устроена в виде пайплайна:

```
SQLite → TopicHistory → DynamicsAnalyzer → ExplainableRec → LLMStub → текст / PDF
```

LLMStub не анализирует оценки сам — только красиво форматирует то, что уже посчитал `DynamicsAnalyzer`.

## Быстрый старт

```bash
# 1. Клонировать
git clone https://github.com/ManagerNotMoney/pedagog.git
cd pedagog

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Запустить
python pedagog.py
```

Зависимости: `openpyxl` (импорт Excel) и `reportlab` (экспорт PDF). Если Excel не нужен — работает и без `openpyxl`.

---

## Сборка EXE

Собирается в один файл без установки Python на целевой машине.

```bash
pip install pyinstaller
pyinstaller pedagog.spec
```

Готовый бинарник появится в `dist/pedagog.exe` (Windows) или `dist/pedagog` (Linux).

> ⚠️ EXE собирается под ту же ОС, на которой запускается сборка. Кросс-компиляция не поддерживается PyInstaller.

## Тесты

```bash
pip install pytest
pytest tests/ -v
```

116 тестов покрывают:
- `DynamicsAnalyzer` — все статусы динамики, расчёт дельты, сортировку рекомендаций
- `ExplainableRec` — приоритеты, поля, форматирование
- `LLMStub` — все типы анализа, граничные случаи
- `PedagogHCA` — публичный API
- `TopicHistory` — свойства, порядок оценок, `__slots__`
- Склонения — работу/дня/недели/месяца с числами 1, 2, 4, 5, 11, 21

## Данные

База данных `pedagog.db` создаётся автоматически при первом запуске рядом с исполняемым файлом. Все данные хранятся локально, никуда не отправляются.

`pedagog.db` добавлен в `.gitignore` — данные учеников в репо не попадают.

## Зависимости

| Пакет | Зачем | Обязательно |
|---|---|---|
| `openpyxl` | Импорт учеников из Excel | Нет |
| `reportlab` | Экспорт отчётов в PDF | Нет |
| `pytest` | Запуск тестов | Только для разработки |

Без `openpyxl` и `reportlab` приложение работает — соответствующие кнопки просто не появляются в UI.
